## Core Principles

- Write simple, readable code. Avoid over-engineering.
- Follow existing codebase patterns exactly. Never invent new conventions.
- Validate every change with tests and type checks before considering it done.
- When uncertain about requirements, ASK before assuming.
- One logical change per commit. Never bundle unrelated changes.
